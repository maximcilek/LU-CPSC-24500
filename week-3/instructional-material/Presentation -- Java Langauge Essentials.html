Conditional logic

Conditionals let your program choose a path. In OOP, you’ll use them to validate state and enforce rules.

Common building blocks

  • Comparisons: < <= > >= == !=
  • Logic: && (AND), || (OR), ! (NOT)
  • Grouping: parentheses ( ... )

Big gotcha

Strings: use .equals() for content equality.

Avoid: role == "admin" (can fail unexpectedly).

if / else (basic)

Use braces for clarity and safety.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

int age = 19;

if (age >= 18) {
  System.out.println("Adult");
} else {
  System.out.println("Minor");
}

Compound conditions

Combine tests with && and ||. Java short-circuits (may skip the right side).

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

int score = 87;
boolean hasExtraCredit = true;

if (score >= 90 || (score >= 85 && hasExtraCredit)) {
  System.out.println("A-range");
} else {
  System.out.println("Not A-range");
}
Try: set hasExtraCredit to false.

String comparisons

.equals() checks content. == checks reference identity.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

String role = "admin";

if (role.equals("admin")) {
  System.out.println("Full access");
} else {
  System.out.println("Limited access");
}

else-if chains

Multiple exclusive paths.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

int tempF = 55;

if (tempF >= 80) {
  System.out.println("Hot");
} else if (tempF >= 60) {
  System.out.println("Warm");
} else if (tempF >= 40) {
  System.out.println("Cool");
} else {
  System.out.println("Cold");
}

Ternary operator

Use ternary for simple value selection (keep it readable).

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

int items = 3;
String label = (items == 1) ? "item" : "items";

System.out.println("You have " + items + " " + label + ".");

switch (classic)

Use break to avoid fall-through (unless you want it).

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

int day = 6; // 1=Mon ... 7=Sun

switch (day) {
  case 6:
  case 7:
    System.out.println("Weekend");
    break;
  default:
    System.out.println("Weekday");
}

switch expression (modern)

Newer Java versions support switch expressions that return a value.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

String grade = "B";

int points = switch (grade) {
  case "A" -> 4;
  case "B" -> 3;
  case "C" -> 2;
  case "D" -> 1;
  default  -> 0;
};

System.out.println(points);

Guard clauses

Handle invalid cases early, then keep the “happy path” simpler.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

public static double discountPrice(double price, boolean isStudent) {
  if (price < 0) {
    throw new IllegalArgumentException("price must be >= 0");
  }

  if (!isStudent) {
    return price;   // guard clause
  }

  return price * 0.9; // 10% off
}

Loops

Loops repeat a block of code. In OOP, you’ll loop over collections of objects and apply behavior.

Pick the right loop

  • for: when you know the number of repeats
  • while: repeat while a condition stays true
  • do-while: run at least once
  • enhanced for: iterate values directly (arrays/collections)

while loop

Good for “unknown count” repetition.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

int n = 5;

while (n > 0) {
  System.out.println(n);
  n--;
}

System.out.println("Blastoff!");

do-while loop

Runs the body first, then checks the condition.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

int attempts = 0;

do {
  attempts++;
  System.out.println("Attempt " + attempts);
} while (attempts < 3);

for loop (counting)

Initialize → condition → update.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

for (int i = 0; i < 5; i++) {
  System.out.println("i = " + i);
}
Common bug: Off-by-one errors (watch < vs <=).

Reverse loop

Counting down is common for indexes.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

for (int i = 4; i >= 0; i--) {
  System.out.println(i);
}

Iterating arrays (index)

Use an index when you need positions.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

int[] nums = {2, 4, 6, 8};

for (int i = 0; i < nums.length; i++) {
  System.out.println("nums[" + i + "] = " + nums[i]);
}

Enhanced for loop

Best when you just need values.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

String[] names = {"Ava", "Ben", "Cal"};

for (String name : names) {
  System.out.println(name.toUpperCase());
}

break and continue

break exits the loop. continue skips to the next iteration.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

for (int i = 1; i <= 10; i++) {
  if (i == 7) break;         // stop early
  if (i % 2 == 1) continue;  // skip odd numbers
  System.out.println(i);
}

Nested loops

Each outer iteration runs the full inner loop.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

for (int row = 1; row <= 3; row++) {
  for (int col = 1; col <= 4; col++) {
    System.out.print("(" + row + "," + col + ") ");
  }
  System.out.println();
}
Performance note: Nested loops can get expensive quickly.

Looping over objects

In OOP, you loop to call methods on objects.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

class Student {
  private final String name;
  Student(String name) { this.name = name; }
  public String getName() { return name; }
}

Student[] roster = { new Student("Ava"), new Student("Ben") };

for (Student s : roster) {
  System.out.println("Hello, " + s.getName());
}

Methods

Methods are named chunks of behavior. In OOP, they define what objects can do.

Method anatomy

  • Modifiers (optional): public, private, static…
  • Return type: like int, String, or void
  • Name + parameters
  • Body in braces

Defining and calling a method

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

public class Main {
  public static void main(String[] args) {
    int result = add(2, 3);
    System.out.println(result);
  }

  public static int add(int a, int b) {
    return a + b;
  }
}

void vs returning a value

Use void when the method’s job is “do something,” not “compute something.”

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

public static void printLine(String msg) {
  System.out.println(msg);
}

public static int square(int n) {
  return n * n;
}

Parameters vs arguments

Parameters are in the definition. Arguments are what you pass at the call site.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

public static void greet(String name) {   // parameter
  System.out.println("Hi, " + name);
}

public static void main(String[] args) {
  greet("Ava");                            // argument
}

Return values & early returns

Returning early can simplify logic (similar to guard clauses).

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

public static boolean isEven(int n) {
  if (n % 2 != 0) {
    return false;
  }
  return true;
}
Refactor: return n % 2 == 0;

Method overloading

Same name, different parameter list (types and/or count).

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

public static int max(int a, int b) {
  return (a > b) ? a : b;
}

public static double max(double a, double b) {
  return (a > b) ? a : b;
}

Varargs (variable arguments)

Varargs lets callers pass 0..N values. Inside the method, it behaves like an array.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

public static int sum(int... nums) {
  int total = 0;
  for (int n : nums) {
    total += n;
  }
  return total;
}

public static void main(String[] args) {
  System.out.println(sum(1, 2, 3));
  System.out.println(sum());
}

Scope (where variables live)

Variables exist only inside the block where they’re declared.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

public static void demo() {
  int x = 10;

  if (x > 5) {
    int y = 99;
    System.out.println(y);
  }

  // System.out.println(y); // ERROR: y is out of scope here
}

Static vs instance methods

Static belongs to the class. Instance belongs to an object.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

class Counter {
  private int count = 0;

  public void increment() {   // instance
    count++;
  }

  public int getCount() {     // instance
    return count;
  }

  public static Counter make() { // static factory method
    return new Counter();
  }
}

Pass-by-value (important!)

Java always passes arguments by value. For objects, the value is the reference.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

class Box { int value; Box(int v) { value = v; } }

public static void change(Box b) {
  b.value = 42;          // changes the object b refers to
  b = new Box(100);      // reassigns local b only
}

public static void main(String[] args) {
  Box box = new Box(1);
  change(box);
  System.out.println(box.value); // prints 42
}

Method design for OOP

Good methods are small, focused, and honest about what they need.

Do

  • Validate inputs (or document assumptions)
  • Keep side effects intentional
  • Extract helpers instead of duplicating code

Avoid

  • Deep nesting (hard to read and debug)
  • “Magic numbers” (use named constants)
  • One method doing “everything”

Exception handling

Exceptions let you handle errors without crashing—or fail fast with clear messages when you must protect correctness.

Keywords

  • throw: raise an exception
  • try/catch: handle exceptions
  • throws: declare a method might throw an exception
  • finally: runs no matter what (cleanup)

try / catch (basic)

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

public static void main(String[] args) {
  try {
    int x = 10 / 0;
    System.out.println(x);
  } catch (ArithmeticException ex) {
    System.out.println("Can't divide by zero!");
  }
}

Multiple catch blocks

Catch the most specific exceptions first.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

try {
  String s = null;
  System.out.println(s.length());
} catch (NullPointerException ex) {
  System.out.println("s was null");
} catch (Exception ex) {
  System.out.println("Something else happened");
}

Multi-catch

Handle different exceptions the same way.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

try {
  Integer.parseInt("not a number");
} catch (NumberFormatException | IllegalArgumentException ex) {
  System.out.println("Bad input: " + ex.getMessage());
}

finally

finally runs whether the try succeeds or fails. Use it for cleanup.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

try {
  System.out.println("Work");
} catch (Exception ex) {
  System.out.println("Handle it");
} finally {
  System.out.println("Always runs");
}

Checked vs unchecked (concept)

Some exceptions must be declared/handled (checked). Others are runtime errors (unchecked).

Checked (must handle/declare)

  • IOException, FileNotFoundException…
  • Often from I/O, external resources

Unchecked (runtime)

  • NullPointerException, IllegalArgumentException…
  • Often logic/validation problems
OOP connection: Validate object state early to prevent runtime crashes later.

throw (fail fast)

Throw exceptions when input violates a method’s contract.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

public static int parsePositiveInt(String s) {
  int n = Integer.parseInt(s);
  if (n <= 0) {
    throw new IllegalArgumentException("Must be positive: " + n);
  }
  return n;
}

throws (declare)

Declare checked exceptions when you want callers to decide how to handle them.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

import java.io.IOException;
import java.nio.file.Files;
import java.nio.file.Path;

public static String readAll(Path path) throws IOException {
  return Files.readString(path);
}

Try-with-resources

Automatically closes resources even if an exception occurs.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public static int countLines(String filename) throws IOException {
  try (BufferedReader br = new BufferedReader(new FileReader(filename))) {
    int count = 0;
    while (br.readLine() != null) {
      count++;
    }
    return count;
  }
}

Custom exception (domain errors)

Create custom exceptions to communicate your domain rules clearly.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

class OverdrawnException extends RuntimeException {
  public OverdrawnException(String message) {
    super(message);
  }
}

class BankAccount {
  private double balance;

  public BankAccount(double initial) {
    if (initial < 0) throw new IllegalArgumentException("initial < 0");
    balance = initial;
  }

  public void withdraw(double amount) {
    if (amount <= 0) throw new IllegalArgumentException("amount <= 0");
    if (amount > balance) throw new OverdrawnException("Not enough funds");
    balance -= amount;
  }
}

Don’t swallow exceptions

Empty catch blocks hide bugs and make debugging painful.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

try {
  // risky code
} catch (Exception ex) {
  // BAD: nothing here
}
Better: handle it meaningfully, or rethrow with context.

Wrap & rethrow with context

Catch a low-level exception and rethrow a clearer one (keeping the cause).

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

try {
  Integer.parseInt(userInput);
} catch (NumberFormatException ex) {
  throw new IllegalArgumentException(
    "Expected a whole number, got: " + userInput,
    ex
  );
}

Putting it together

A small example that uses: validation + loop + method + exceptions.

        
        
                

         
         
                  

          Java
         
         
                  

         
         
                  

          
         
         
                  

        
        
                

public static double averageNonNegative(int[] nums) {
  if (nums == null) {
    throw new IllegalArgumentException("nums is null");
  }
  if (nums.length == 0) {
    throw new IllegalArgumentException("nums is empty");
  }

  int sum = 0;

  for (int n : nums) {
    if (n < 0) {
      throw new IllegalArgumentException("negative value: " + n);
    }
    sum += n;
  }

  return (double) sum / nums.length;
}
